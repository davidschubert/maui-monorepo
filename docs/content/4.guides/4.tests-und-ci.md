## Test-Ebenen

| Ebene | Werkzeug | Wo |
| --- | --- | --- |
| Unit | Vitest (`pnpm -r test`) | pro Package (z. B. OKLCH-Ramp in `themes`) |
| E2E | Playwright | `apps/comments` (Referenz-App) |
| Visual | Playwright-Screenshots | `themes` — gegen die deterministische `/visual`-Seite |
| Statisch | ESLint, vue-tsc, `check:manifests` | Root / CI |

## E2E-Suite

- Base-URL per `PW_BASE_URL` überschreibbar — so laufen parallele Dev-Sessions nebeneinander.
- Visual-Tests zielen auf `/visual` — **nie Live-Daten screenshotten**.
- RBAC-Flows auch mit einem least-privileged User (ohne Labels) testen.

::note
Bekanntes Verhalten: Playwright-Worker können nach grünen Tests ~5 min im Teardown hängen — nicht killen; Ausgabe ungepuffert via `npx` starten.
::

## CI

- **lint-Job:** ESLint + `pnpm check:manifests` (Manifest-/extends-/deps-Konsistenz, Destruktiv-Guard).
- **e2e.yml:** fährt eine echte **Wegwerf-Appwrite** hoch (`ci/appwrite` + `scripts/ci/appwrite-setup.mjs` → `bootstrap --seed`) und lässt die volle Suite inkl. Realtime laufen.
- Release-Please erzeugt Release-PRs aus Conventional Commits.

## Lokal vor dem Push

```bash
nvm use 22
pnpm lint && pnpm typecheck && pnpm -r test && pnpm check:manifests
```

::tip
Projekt-Prinzip: **erst grün, dann nächster Schritt** — inkrementell verifizieren statt am Ende alles auf einmal debuggen.
::
