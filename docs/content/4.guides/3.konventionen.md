## Vue & UI

- `<script setup lang="ts">`, Nuxt-UI-Komponenten bevorzugen.
- **Auth-Formulare:** `UAuthForm` ist die optische Vorlage — Login/Register/OTP sind bewusst eigene `UForm`-Implementierungen (2-Schritt-OTP, Security-Phrase, geteilter E-Mail-State, AGB-Gate). Details: `docs/AUTH-FORMS.md`.
- Nuxt-UI/Reka-Stolperfallen: `SelectItem` verbietet `''`-Werte; kein SSR/Client-Branch-Swap (Hydration!); offene Modals nie per `v-if` unmounten.
- Pinia: `defineStore` im Composition Style.

## Formulare & Validierung

- **Zod für alle Formulare** — Schemas als `create*Schema(t)`-Factories (bekommen die i18n-`t`-Funktion).
- User-facing Strings **immer** über i18n-Keys — keine hartcodierten Strings in Markup oder Toasts.
- `@` in Locale-Messages als `{'@'}` escapen.

## i18n

Strategie `prefix_except_default`: `en` als Default ohne Prefix, `de` unter `/de/*`; `detectBrowserLanguage` mit `redirectOn: 'all'` — bewusst **ohne** `fallbackLocale` (signal-lose Requests wie Crawler behalten die URL-Locale, sonst gäbe es EN-Content unter `/de/*`).

::caution
Interne Links und Redirects **immer** über `localePath()` — auch in Middleware (`useLocalePath()('/...')}`). Sonst geht der Locale-Prefix verloren.
::

SEO: `useLocaleHead` in den App-`app.vue` liefert hreflang/canonical/og:locale; absolute URLs via `NUXT_PUBLIC_I18N_BASE_URL`.

## Server-Code

- `createError` mit `status`/`statusText` (nicht `statusCode`/`statusMessage`).
- Keine Appwrite-Fehlerdetails an Clients leaken.
- Explizites `Query.limit()`, SDK-Generics (`listRows<T>()`).
- CRUD nur über `server/api/*` — siehe [Appwrite](/architektur/appwrite).

## TypeScript & Dependencies

- TypeScript **strict**, kein `any`, vollständige Dateien, keine Spekulation.
- pnpm-Catalog: Versionen zentral in `pnpm-workspace.yaml`, Packages referenzieren `"catalog:"` — geteilte Deps auch in der App-`package.json` eintragen.

## Git

- **Conventional Commits**; Breaking Changes am Core mit `BREAKING CHANGE(core):`-Prefix.
- Core-Änderungen in **eigenem Commit**; vor einem Core-Update alle Apps lokal starten.
- Release-Automatik: Release-Please; Changelog-Entwürfe via `pnpm changelog:draft`.
