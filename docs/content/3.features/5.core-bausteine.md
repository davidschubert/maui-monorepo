## Config-Gates (`maui.*`)

Querschnitts-Features werden über `app.config.ts` im Namespace `maui.*` gegatet. Die Config wird **tief gemergt** — die App überschreibt nur, was nötig ist. Core-Default ist fast immer **aus**:

| Gate | Zweck |
| --- | --- |
| `maui.analytics` / `maui.consent` | Tracking + Consent-Banner (App aktiviert explizit) |
| `maui.observability` | strukturierte JSON-5xx-Logs am zentralen `server/error.ts` + Client-Error-Inbox (`POST /api/telemetry/error`, rate-limited); Sentry-Andockpunkt in `logEvent.ts` |
| `maui.auth.*` | OAuth-`providers`, `termsUrl` (AGB-Pflicht), `otp` |
| `maui.ai` | KI-Transport (enabled/model/baseUrl) |
| `maui.comments.embed` | Embed-Freigabe + `allowedOrigins` |
| `maui.comments.autoHideReports` | Auto-Hide-Schwelle (0 = aus) |
| `maui.admin.modules` | Modul-Registry der Dashboard-Nav |

## KI: `aiComplete()` / `aiCompleteJson()`

`core/server/utils/aiComplete.ts` ist **ein** Transport für OpenAI-kompatible APIs (Default: OpenRouter):

- Gate `maui.ai` (Core-Default aus) + server-only `NUXT_AI_KEY`.
- Der Transport ist **policy-frei** — Gates und Antwort-Klemmung liegen beim Konsumenten.
- Laufzeit-Override-Kette für das Modell: `app_config.ticketsAiModel` → `app_config.aiModel` (Admin-Config-Seite, `getEffectiveAiConfig`) → `maui.tickets.ai` → `maui.ai`.
- Konsumenten: Ticket-Triage, Moderations-Assist (Kommentare + Posts) — immer advisory.

## E-Mail: `sendMail()` + Benachrichtigungen

- `sendMail()` (core `mailer.ts`, nodemailer, `NUXT_SMTP_*`) — leerer Host = aus; lokal Mailpit auf `localhost:1025`.
- `notify()` hat einen Opt-in-E-Mail-Zweig: `prefs.emailNotifications` `off | instant | digest` (Default off, einstellbar unter Settings → Benachrichtigungen; Mail-Sprache = `prefs.emailLocale`).
- **Digest-Sweep:** Kandidaten kommen aus ungelesenen `notifications`-Rows (kein User-Scan), max. 1 Mail/Tag (`prefs.emailDigestLastAt`, merge!), Intervall-Plugin alle 30 min + `POST /api/notifications/run-digest` (`system.manage`).

## GDPR: Export & Löschung

Feature Layer mit User-Daten **müssen** einen Contributor registrieren (`registerUserDataContributor` aus `core/server/utils/userData.ts`, per Nitro-Plugin `server/plugins/user-data.ts`):

- Core orchestriert Export und Löschung.
- `deleteUserCompletely`: Snapshot → Sperren → Contributors → `users.delete` **nur bei Voll-Erfolg**.

## Rate-Limiting & Sicherheit

- `rate-limit.ts`-Middleware in core schützt sensible Endpoints.
- `csrf-origin.ts` (`maui.security.csrfOriginCheck`) — Pflicht, sobald Partitioned-Cookies (Embed E2) kommen.
- Grundsatz: keine Appwrite-Fehlerdetails an Clients leaken.

## Microcache

`createMicrocache()` nur für **user-agnostische GETs** (Gast-Kommentare Seite 1, Changelog, `/api/stats`) — nie Session-Daten cachen, kein SSR-Seiten-SWR.
