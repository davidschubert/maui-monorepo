## Fundament

| Layer | Verantwortung |
| --- | --- |
| `core` | Auth (Login/Register/OTP/OAuth), Appwrite-Clients, UI-Fundament, i18n-Skeleton, Notifications/E-Mail, KI-Transport, GDPR-Orchestrierung, Microcache, Observability. Besitzt **keine** Tables. |
| `system` | Infra-Tabellen: `audit_logs`, `app_config`, `notifications`; system-Migrationen (u. a. für Themes-Tables). |
| `moderation` | Meldungen (Reports), Eskalations-Registry, Verträge für Resolve/Assist/Bulk. Fundament-nah — Feature-Layer docken an. |

## Feature Layers

| Layer | Verantwortung |
| --- | --- |
| `themes` | Theme-Studio (`/dashboard/themes`), Custom Themes aus einer Basisfarbe (OKLCH-Ramp), Schriften-Verwaltung inkl. WOFF2-Upload. Rein visuell — importiert weder Appwrite noch andere Layer. |
| `comments` | Kommentare inkl. Zweiphasen-Hide, Cascade, Auto-Hide ab N Meldungen, Embed (Read-only-MVP), Thread-Presence. |
| `posts` | Beiträge inkl. Moderations-Assist. |
| `admin` | Dashboard-Gerüst: Modul-Registry (`maui.admin.modules`), globale Suche, Feature-Katalog + Laufzeit-Toggles, Changelog-Workflow. |
| `billing` | Stripe-Integration (Webhook, Abos). |
| `tickets` | Support-Tickets inkl. KI-Triage und Benachrichtigungen. |
| `events` | Termine/Veranstaltungen. |
| `feedback` | Feedback-Erfassung. |
| `courses` | Kurse. |
| `activity` | Aktivitäts-Feed (früher „feed"). |

Die Detailseiten dieses Kapitels behandeln die drei größten Welten — [Themes](/features/themes), [Kommentare & Moderation](/features/comments-und-moderation), [Admin](/features/admin) — sowie die [Core-Bausteine](/features/core-bausteine) (KI, E-Mail, Embed, GDPR, Config-Gates).

## Feature-Wahl pro Site

Welche Layer eine App bekommt, entscheidet ihr [Site-Manifest](/architektur/manifeste). Zur Laufzeit kommen Config-Gates (`maui.*`) und Admin-Toggles dazu — Grundgerüst-Features zeigen im Feature-Katalog bewusst „Immer aktiv" statt eines toten Schalters.
