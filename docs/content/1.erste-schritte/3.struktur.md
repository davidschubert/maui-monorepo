## Verzeichnis-Layout

```
maui-monorepo/
├── packages/
│   ├── core/            # Fundament: Auth, Appwrite-Clients, UI-Basis, Utilities
│   ├── system/          # Fundament: Infra-Tabellen (audit_logs, app_config, notifications)
│   ├── themes/          # Theme-Studio, Custom Themes & Fonts
│   ├── comments/        # Kommentare inkl. Moderations-Logik
│   ├── posts/           # Beiträge
│   ├── admin/           # Admin-Dashboard (Nav-Registry, Suche, Changelog)
│   ├── moderation/      # Meldungen/Eskalation (Fundament-nah)
│   ├── billing/         # Stripe-Integration
│   ├── tickets/         # Support-Tickets (inkl. KI-Triage)
│   ├── events/ feedback/ courses/ activity/
│   └── …                # jeder Layer: feature.manifest.ts
├── apps/
│   ├── _template/       # Scaffold-Vorlage für create-site
│   ├── reddit-comments/ # Referenz-App (E2E-Suite)
│   └── …                # jede App: site.manifest.ts
├── docs/                # Markdown-Wissensbasis + DIESE Docs-App (Nuxt Content)
├── scripts/
│   ├── migrate.mjs          # zentraler Migrations-Runner
│   ├── check-manifests.mjs  # Manifest-/extends-/deps-Konsistenz (CI)
│   ├── create-site.mjs      # neue Site in einem Befehl
│   └── ci/                  # Wegwerf-Appwrite für die E2E-CI
└── pnpm-workspace.yaml      # Workspaces + zentraler Dependency-Catalog
```

## Workspaces & Dependency-Catalog

Workspaces: `packages/*`, `apps/*` und `docs`. Geteilte Dependency-Versionen liegen **zentral** im pnpm-Catalog (`pnpm-workspace.yaml`); die `package.json` der Pakete referenziert nur `"catalog:"`:

```json [packages/x/package.json]
{
  "dependencies": {
    "nuxt": "catalog:",
    "@nuxt/ui": "catalog:"
  }
}
```

## Aufbau eines Layers

Jeder Layer ist ein vollwertiges Nuxt-Layer-Package:

```
packages/<layer>/
├── feature.manifest.ts   # key/tier/requires/Katalog-Texte (nur `import type`!)
├── app/                  # Components, Pages, Composables, app.config.ts
├── server/               # api/, utils/, plugins/, middleware/
├── shared/types/         # Domain-Types (Server + App sichtbar)
├── i18n/locales/         # de.json, en.json
├── scripts/migrations/   # nur wenn der Layer eigene Tables besitzt
└── nuxt.config.ts
```

Wichtige Konventionen:

- **Relative Pfade im Layer** — kein `~/` oder `@/`.
- `app.config.ts` liegt in `app/` — im Package-Root wird sie stillschweigend ignoriert.
- Domain-Types nach `shared/types/` — in `app/types/` sieht der Server sie nicht.
- Pinia-Stores der Layer via `imports.dirs` registrieren (werden nicht auto-gescannt).
- `error.vue` wird nicht aus Layern aufgelöst: Markup lebt in `CoreErrorPage`, jede App hat eine dünne `app/error.vue` als Wrapper.

## Sonderfall: dieser Docs-Bereich

`docs/` ist **kein** Maui-Layer und keine App unter `apps/` — bewusst: Der Manifest-Check verlangt für alles unter `apps/` ein Site-Manifest samt Appwrite-Setup. Die Docs-App ist eine eigenständige Nuxt-App (Nuxt UI + Nuxt Content) ohne Appwrite, die die Markdown-Inhalte unter `docs/content/` rendert.
